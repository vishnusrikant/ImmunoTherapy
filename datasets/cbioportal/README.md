# cBioPortal — Immunotherapy Clinical + Genomic Data

Public cancer genomics studies with immunotherapy cohorts, downloaded from the cBioPortal REST API (https://www.cbioportal.org/api) on 2026-04-15.

## Why This Source

cBioPortal complements FAERS and ImmPort in three ways:

1. **Scale beyond ImmPort** — 1,218 immunotherapy patients across 7 studies (vs. 86 on ImmPort)
2. **Pre-treatment clinical features FAERS lacks** — LDH, ECOG, TMB, metastasis sites, steroid use, prior therapy lines
3. **Survival + response outcomes** — OS / PFS months and RECIST-style response labels for validation

**Caveat:** cBioPortal studies focus on treatment RESPONSE (responder / non-responder) rather than adverse-event severity. There are **no irAE columns**. Use cBioPortal to enrich pre-treatment features; continue using FAERS for severity labels.

## Access

No authentication required — all endpoints are public.

```
GET https://www.cbioportal.org/api/studies/{study_id}
GET https://www.cbioportal.org/api/studies/{study_id}/patients
GET https://www.cbioportal.org/api/studies/{study_id}/samples
GET https://www.cbioportal.org/api/studies/{study_id}/clinical-attributes
GET https://www.cbioportal.org/api/studies/{study_id}/clinical-data?clinicalDataType=PATIENT
GET https://www.cbioportal.org/api/studies/{study_id}/clinical-data?clinicalDataType=SAMPLE
```

Clinical data is returned in long format (one row per patient × attribute). The download script pivots to wide format (one row per patient).

### Response vocabulary caveat

The response column mixes two conventions:

- **Binary** (most studies): `TRUE` = responder (CR/PR), `FALSE` = non-responder (SD/PD)
- **RECIST** (`mel_dfci_2019`): `Complete Response`, `Partial Response`, `Stable Disease`, `Progressive Disease`, `Mixed Response`

Normalize before modeling — e.g. map `CR`+`PR` → responder, `SD`+`PD` → non-responder.

## Known Limitations

1. **No adverse-event columns.** cBioPortal does not record irAEs. Use FAERS for severity labels.
2. **Age coverage is poor (27%).** Many studies report only age groups or omit age entirely.
3. **LDH / ECOG / metastasis detail exists only in `mel_dfci_2019`.** For cross-study models, treat these as optional features with explicit missingness indicators.
4. **Response vocabulary is heterogeneous.** See caveat above.
5. **Not a severity prediction dataset.** These are response-prediction cohorts; use them to enrich features, not as a replacement for FAERS.

## How To Use With The Severity Model

**Primary use case: augment the feature set for a subset of FAERS reports.**

- LDH elevated, ECOG ≥ 2, brain/liver metastases, and baseline steroid use are all validated correlates of irAE risk
- TMB correlates with checkpoint inhibitor response and — indirectly — with irAE likelihood
- Train a severity model on FAERS; separately, train a response predictor on cBioPortal and compare which features matter for each outcome

**Secondary use case: validation cohort.**

- For melanoma / NSCLC / bladder / RCC patients where ImmPort does not overlap, cBioPortal provides a second pre-treatment feature set to sanity-check FAERS-derived predictions.
